# _RETEX — crossroads swift

Retrospectives — what we learned after shipping. Distinct from `_INCIDENTS/` (what broke) and `_RESEARCH/` (what we might explore).

## Shape

- One file per retro: `YYYY-MM-DD-topic.md`
- Triggered by: releases, milestones, user feedback cycles, quarterly reviews

## Template

```markdown
# YYYY-MM-DD — <topic>

## Context

<What shipped or happened>

## What worked

<Specific, not generic>

## What didn't work

<Specific, including assumptions that broke>

## What we'll do differently

<Concrete process / code / product changes>

## Patterns to promote

<If a lesson is reusable, link from here to 00_MAIN/patterns/>
```

## Rule

A retro that never changes behavior is decorative. Every retro ends with at least one concrete "we'll do X differently" — and that X is tracked somewhere actionable.
